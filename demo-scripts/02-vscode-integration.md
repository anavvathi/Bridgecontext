# Demo Video 2: ChatGPT to VS Code (2 minutes)

## Target Audience
Developers who use ChatGPT for coding

## Key Message
"Stop copy-pasting between ChatGPT and VS Code"

## Script

### [0:00-0:20] THE OLD WAY (Pain Point)
**Visual**: Screen recording of typical workflow
**Narration**:
> "Here's how most developers use ChatGPT..."

**Show step-by-step** (sped up 2x):
1. Type question in ChatGPT
2. Wait for response
3. Select all code
4. Ctrl+C (copy)
5. Alt+Tab to VS Code
6. Ctrl+V (paste)
7. Realize you need more context
8. Alt+Tab back to ChatGPT
9. Scroll to find relevant part
10. Copy again
11. Repeat...

**On-screen text**: "Sound exhausting?"

---

### [0:20-0:50] THE NEW WAY (Solution)
**Visual**: Clean demo of BridgeContext workflow
**Narration**:
> "Install BridgeContext from the Chrome Web Store."

**Show**:
1. Chrome Web Store page
2. Click "Add to Chrome"
3. Extension installed

**Narration**:
> "Now when you're in ChatGPT, press Alt+Shift+I."

**Show**:
1. ChatGPT conversation about React hooks
2. Press Alt+Shift+I
3. VS Code opens with BridgeContext sidebar
4. Full conversation appears in sidebar
5. Click any message
6. Code inserts at cursor position

**On-screen text**: "10 seconds. No copy-paste."

---

### [0:50-1:30] ADVANCED FEATURES
**Visual**: Feature showcase
**Narration**:
> "Works with Claude, Gemini, DeepSeek, and more."

**Show**: Quick cuts of different AI platforms

**Narration**:
> "Supports VS Code, Cursor, JetBrains, Neovim."

**Show**: IDE logos

**Narration**:
> "Save context packs for reuse."

**Show**: 
1. Create context pack with coding style guide
2. Inject into new ChatGPT conversation
3. ChatGPT responds with consistent style

**Narration**:
> "Search through all your saved contexts."

**Show**:
1. Search bar in extension popup
2. Type "React hooks"
3. Relevant contexts appear
4. Click to inject

**On-screen text**: "Your AI memory, organized."

---

### [1:30-1:50] PRIVACY & SECURITY
**Visual**: Privacy features
**Narration**:
> "100% local. Your code never leaves your machine."

**Show**:
1. Extension popup showing "Local-First" badge
2. No network requests in DevTools
3. Data stored in browser's local storage

**On-screen text**: 
- "No Cloud"
- "No Tracking"
- "Open Source"

---

### [1:50-2:00] CALL TO ACTION
**Visual**: Chrome Web Store badge, GitHub logo
**Narration**:
> "Free and open source. Link in description. Stop copy-pasting. Start bridging."

**On-screen text**:
- "Get BridgeContext"
- "Chrome Web Store"
- "github.com/anavvathi/Bridgecontext"
